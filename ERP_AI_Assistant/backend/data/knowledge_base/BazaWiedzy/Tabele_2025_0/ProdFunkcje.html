 SQL_IDENTITY
PFU:Kod CSTRING(41) Kod gniazda roboczego

PFU:Opis CSTRING(513) Opis gniazda roboczego
PFU:Oddzial LONG Numer oddziału lub brak
PFU:Archiwalny BYTE Czy archiwalny
PFU:Zmienna BYTE Czy obiekty mogą byc zmieniane dla różnych terminów
PFU:Planowana BYTE dobierać zasoby podczas planowania 0-nigdy, 1- szczegółowego, 2- zgrubnego, 3-zawsze
PFU:Typ BYTE 1 - gniazdo robocze w module produkcja, 2 - w module kadry, 3 - w module produkcja i kadry
PFU:CzyZrownoleglac BYTE

PFU:KontynuacjaPracy BYTE

PFU:PlanujStrategicznie BYTE Czy gniazdo robocze należy uwzględniać w planowaniu strategicznym (czyli w planowaniu obciążenia gniazd roboczych)
0 - Gniazdo robocze nie będzie uwzględniane przy planowaniu strategicznym. 1 - Gniazdo robocze należy uwzględniać w planowaniu strategicznym (czyli w planowaniu obciążenia gniazd roboczych)
PFU:ZdolnoscWOkresie DECIMAL(11,2) Zdolność produkcyjna gniazda roboczego
PFU:ZdolnoscWOkresieJedn BYTE Jednostka dla wartości: Zdolność w okresie
0 - minuty, 1 - godziny, 2 - dni.
PFU:DopuszczalneObciazenie SHORT Procentowe obciążenie brane pod uwagę przy szacowaniu obciążenia gniazda roboczego
PFU:TypGniazda LONG

PFU:Zablokowana BYTE decyduje o tym, czy zasób będzie podmieniany przy przeplanowaniu zlecenia
PFU:DomyslnaRobociznaWBudzetach BYTE Domyślna robocizna w budżetach
PFU:StawkaCzas DECIMAL(15,4) Stawka za czas wykorzystania
PFU:StawkaCzasAtr LONG Klasa atrybutu ze stawką za czas
PFU:StawkaCzasM LONG Mianownik stawki za czas w sekundach
PFU:StawkaCzasMJedn BYTE Jednostka czasu mianownika
Wartości: 0 - sekundy 1 - minuty 2 - godziny 3 - dni 4 - tygodnie
PFU:UwzglednijWNrSeryjnym BYTE Czy funkcja ma być uwzględniana w nr seryjnym
PFU:TenSamZasobWPartiach BYTE Czy gniazdo ma wykorzystywać ten sam zasób w każdej partii operacji
